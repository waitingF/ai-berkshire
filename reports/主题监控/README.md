# 主题监控（L0）使用说明

**层级定位**：主题池的周度/双周度监控层。只回答三件事——景气方向、是否拥挤、要不要升维。  
**不做**：个股终选、估值定价、四大师长文（那是 L1/L2）。

本目录产出用于学习研究，不构成投资建议。

---

## 文件约定

| 文件 | 规则 |
|------|------|
| `README.md` | 本说明；字段与升维规则变更时更新 |
| `主题池-看板-YYYYMMDD.md` | 当期看板；**每周新建一份**，不覆盖旧文件 |
| 「当前态」 | 目录内**日期最新**的那份看板 |

命名示例：`主题池-看板-20260716.md`

建议流程：复制上一期看板 → 改文件名日期与页眉 → 只改本周变化行 → 提交。

---

## 监控单元怎么拆

- 九大主题是**分组**，不是一行糊弄完。
- 看板按**可独立观察的子赛道**拆行（通常 20–35 行）。
- 子赛道过宽（如「整个 AI」）禁止作为唯一监控行；必须拆到可验证环节。

### 跟踪层级

| 层级 | 含义 | 精力 |
|------|------|------|
| **主跟踪** | 仓位相关或本季度重点 | 每周填证据 |
| **副跟踪** | 有信号再加码 | 双周扫一眼 |
| **冷藏** | 季度复盘是否升级 | 默认不周更 |
| **红利单独表** | 防守/现金流主题 | 与成长主题分表，勿混用成长打分 |

默认切分（可改，改在当期看板页眉注明）：

- 主跟踪：AI/数智化、半导体自主可控
- 副跟踪：新能源与新型电力系统、化工/新材料、机器人/具身智能
- 冷藏：低空/航天/北斗、安全韧性、医药医疗与银发、红利与央国企（红利用单独表轻量更新）

## 字段定义

| 字段 | 取值 / 写法 | 硬性要求 |
|------|-------------|---------|
| 主题 | 九大类之一 | 必填 |
| 子赛道 | 可监控单元名称 | 必填 |
| 层级 | 主跟踪 / 副跟踪 / 冷藏 | 必填 |
| 景气 | ↑ / → / ↓ | 必填；无证据则写「待判」并说明 |
| 周期位置 | 早期 / 扩张 / 成熟 / 过剩 | 化工、新能源、部分制造**必填**；其余可「不适用」 |
| 关键证据 | 本周/本期**已发生**事实 | 禁止用预测顶替；估计值标「估计」；附来源或「待补」 |
| 拥挤度 | 低 / 中 / 高 | 粗判即可（成交、涨幅、叙事密度） |
| 价格战风险 | 是 / 否 / 部分环节 | 新能源中游、部分化工、部分制造重点盯 |
| 升维建议 | 保持 L0 / 触发 L1 / 触发 L2 / L1已完成待L2 | 必填 |
| 证伪信号 | 预先写死的反转条件 | 必填；避免事后合理化 |
| 关联报告 | `reports/` 相对路径 | 有则链；无则「—」 |

### 景气三信号（填「关键证据」时优先覆盖）

1. **订单/需求**：大客户签约、capex 指引、招标中标、装机/开工  
2. **价格/供需**：产品价格、库存、交期、开工率  
3. **资金/拥挤**：板块成交额、ETF 申赎、短端涨幅（仅作拥挤参考，不作逻辑证明）

### 写作纪律（与项目投研原则一致）

- 区分事实与观点；观点斜体或标注「观点」
- 不预设立场；不确定就写「不确定」或「待补」
- 每个升维建议尽量附一句反面论据（「但另一方面…」）

## 升维规则（L0 → L1 → L2）

### 保持 L0（默认）

- 证据不足、信号互相矛盾，或主题仍处冷藏且无仓位需求

### 触发 L1（结构深研）

满足**任一**即可开 L1（`/industry-research` 或 `/bottleneck-hunter`）：

1. 连续 2–4 周景气信号**同向**强化  
2. 出现「已发生」的大额验证事件（签约、政策落地、明确产能紧张证据）  
3. 准备真实配仓，需要先看清产业链/瓶颈与价格战环节  

L1 产出：`reports/{主题}-industry-{YYYYMMDD}.md`（AI 类可放 `reports/AI产业研究/`）  
L1 **不强制**终选 3 只股票；要回答：逻辑链是否成立、钱沉淀在哪一环、哪些环节谨慎。

### 触发 L2（选股漏斗）

- L1 结论为「值得配仓」且准备下单 → `/industry-funnel`  
- 重仓前再 `/investment-team` 或 `/investment-checklist`  
- 买入后用 `/thesis-tracker` 维护

### 降维 / 冷藏

- 证伪信号触发 → 升维建议改为「保持 L0」或「冷藏」，并在证据栏写清事实  
- 价格战确认且无差异化环节 → 主跟踪可降为副跟踪；核心仓不进该环节

## 每周操作清单（约 30–60 分钟/主跟踪）

1. 复制上期 `主题池-看板-*.md`，改日期  
2. 只更新**有变化**的行；无变化写「较上期无新增事实」  
3. 检查主跟踪行的证伪信号是否触碰  
4. 若有行改为「触发 L1/L2」，在看板页眉「本周动作」列出  
5. （可选）对准备配仓的主题开 L1/L2 skill

## 首版之后的维护

- 子赛道增减：在当期看板改表，并在页眉「结构变更」记一笔  
- 字段增减：先改本 README，再改新一期看板  
- 推送 GitHub：只推最终看板与 L1/L2 报告；勿推半成品底稿
