## AI研究偏见自觉

### 信息丰富度评级：A级（信息充裕）

IBM 上市逾百年，卖方覆盖密集，10-K/季报/投资者简报齐全，Macrotrends、StockAnalysis、Yahoo 等第三方数据完备。

**AI研究陷阱**：共识过强，输出易趋同于“混合云+AI 再评级”或“股息奶牛”叙事；在 A 级标的上，资料越多越容易产生虚假安全感。

**应对策略**：重点做反面检验——聪明人为什么在 \$290 附近不追高？Q2 初值错配是一次性 capex 挤出，还是 Software 增速结构性下移？Confluent / HashiCorp 大额商誉与债务是否被低估？

**偏见自查**：
- [x] “确定性”感受主要来自资料充裕，不等于 Q2 后全年指引已定——7/22 前存在显著信息缺口
- [x] 若资料量减半，生意本质（主框锁定 + 高毛利软件 + 慢速咨询）结论大致不变；“便宜/贵”结论高度依赖现价
- [x] 分析与市场共识（hybrid cloud / watsonx / Red Hat）高度重叠——本报告独立价值在四大师框架下的失败路径与价格纪律，而非独家信息
- [x] 不存在 C 级“资料少但生意好”陷阱；相反风险是资料太多掩盖执行波动

### AI研究局限性声明

财务数据来自 IBM IR / SEC 10-K、Macrotrends、StockAnalysis，经 `financial_rigor.py` 交叉验证。Q2'26 仅为**初步数据**，最终数字可能微调。公有云份额、watsonx 真实 ARR、客户流失微观数据无法从公开资料精确还原。所有涉及未来增速与估值的判断均为**观点/推测**，须与事实区分。

### 一句话定义

**IBM 的本质是：向大型受监管企业出售“混合云平台软件 + 任务关键主机（IBM Z）+ 落地咨询”，用高切换成本锁定核心系统，再用 Red Hat / watsonx / 自动化软件抽 recurring 利润。**

## 第四步：逆向思考与风险清单 — 芒格“反过来想”

### 失败路径表

| 路径 | 概率（观点） | 影响 | 说明 |
|------|-------------|------|------|
| Software 增速结构性下移（非一次性） | 中 | 高 | Q2 Software 仅 +5%；若 ARR 失速，PE 压缩 |
| z17 周期后 Infrastructure 持续拖累 | 高（周期性） | 中 | 历史主框周期常见；公司已提示低个位数下滑，但 Q2 更差 |
| AI 工具压缩 Consulting 工时/定价 | 中高 | 中–高 | GenAI book 中咨询占比或偏高（次级估计） |
| 大额并购整合失败 / 商誉减值 | 中 | 中 | Confluent ~\$11B EV；HashiCorp ~\$6.4B |
| 净债务上升 + 利率/评级压力 | 中 | 中 | Q1'26 总债务 \$66.4B；股息优先限制回购缓冲 |
| 公有云/AI 分发输给 Microsoft | 已发生 | 中 | 错位竞争可共存，但限制上限故事 |
| 养老金再波动 | 低–中 | 低–中 | 2024 已有大额 settlement，风险未归零 |

### 历史类比

- **成功路径**：类似“从硬件巨头转向平台软件 + 服务”的缓慢重生（非 NVIDIA 式指数曲线）。  
- **失败路径**：若软件增长叙事破裂，可能回落至“高股息、低增长 IT 服务商”估值带（历史 IBM 长期 PE 中枢更低）。

### 空方核心论点（汇总）

1. 指引仅 **恒定货币收入 >5%**，崩盘前 25x+ PE 定价过高。  
2. Q2 显示执行脆弱：大单延期 + capex 挤出即可打穿季度。  
3. GenAI book 若以咨询为主，难支撑软件倍数。  
4. 债务 + 并购商誉抬高下行风险。

### 芒格式追问

> **我最可能在哪里犯错？**  
> 把 Q2 一次性归因为“供应链噪音”，而忽略 **Software +5% 可能是需求质量变差的早期信号**；或反过来，在恐慌中低估 Red Hat / ARR 的韧性。聪明人做空/不买的理由：用软件估值买了一家仍有周期硬件与慢速咨询的组合公司。
