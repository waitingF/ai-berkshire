# 腾讯混元 Hy3 发布核实与真实水平评估

**日期**：2026-07-07（正式版发布次日）
**研究方法**：多路并行网络检索 → 抓取一手信源 → 逐条声明三票对抗核实（2/3 反驳即剔除），共核实 20 余条声明，4 条未通过核实被剔除。

---

## 一、结论摘要

1. **消息属实**。《腾讯混元Hy3发布：Agent能力和产品体验跃升》是新华网 2026 年 7 月 6 日的报道原题，Hy3 正式版当日发布，此前 4 月 23 日已开源预览版。正式版模型卡已上线 Hugging Face 官方 tencent 组织，Apache 2.0 许可，权重开放可下载。
2. **但"跃升"的全部量化依据来自腾讯内部评测**，无第三方背书。幻觉率减半、多轮对话问题率减半、盲测胜过 GLM5.1 等核心数据，评委是腾讯内部员工、场景是腾讯自家产品。
3. **真实水平定位**：国内开源第一梯队的偏后位置，同尺寸（激活 21B）性价比突出，代码与数学能力是强项；与国际闭源前沿（Claude、GPT、Gemini）存在明确差距。唯一可查的第三方独立评测（Artificial Analysis，针对 4 月预览版）给出"明显高于平均、但非顶尖"的定位：93 个可比模型中排第 22。
4. **正式版的成绩（SWE-bench Verified 78 分等）目前完全依赖腾讯自报**，发布仅一天，尚无任何第三方复测。

## 二、发布事实核实（置信度：高）

| 事项 | 核实结果 |
|---|---|
| 发布时间 | 正式版 2026-07-06；预览版 2026-04-23 开源 |
| 消息来源 | 新华网原文报道，证券时报、腾讯新闻、钛媒体等多家媒体同日交叉印证 |
| 开源情况 | 预览版与正式版均上线 Hugging Face / GitHub / ModelScope / GitCode，Apache 2.0 |
| 正式版与预览版关系 | 官方模型卡称正式版是在预览版基础上收集 50 多个产品的反馈、扩大后训练规模而来 |

发布路径清晰：4 月底预览版 → 7 月 6 日正式版，不存在"炒冷饭"或虚假发布的问题。

## 三、模型规格（开源权重可直接验证，置信度：高)

- 快慢思考融合的混合专家（MoE）架构
- 总参数 295B，激活参数 21B，另含 3.8B 多词元预测层参数
- 192 个专家、每次激活 8 个，80 层 transformer
- 最长 256K 上下文
- 预览版与正式版架构一致

**要点**：激活参数只有 21B，属于"大总参、小激活"路线，推理成本低。官方"比肩参数规模 2–5 倍旗舰模型"的说法，卖点本质是**性价比**而非绝对能力。

## 四、官方自报成绩与主张（均为厂商口径）

### 基准测试（正式版，自报）

| 基准 | 正式版 | 预览版（4月） |
|---|---|---|
| SWE-bench Verified | 78 | 74.4 |
| GPQA Diamond | 90.4 | 87.2 |
| SWE-bench Pro | 57.9 | — |
| HLE（人类最后考试） | 53.2 | — |
| Terminal-Bench 2.0 | — | 54.4 |

预览版到正式版有真实提升（74.4 → 78），但部分 Agent 基准官方只给图片对比、不给数值表格。

### Agent 能力主张（自报，未经外部复现）

- 可稳定支撑长达 **495 步**的复杂 Agent 工作流（数字来自腾讯自家产品环境）
- 推理效率较前代提升 40%
- 内部产品实测：CodeBuddy/WorkBuddy 首字时延降 54%、端到端响应降 47%
- 真实场景幻觉率 12.5% → 5.4%，多轮对话问题率 17.4% → 7.9%
- 内部 270 位专家盲测：Hy3 均分 2.67/4，优于 GLM5.1 的 2.51/4（评委为腾讯员工）

**另有 4 条产品落地指标（WorkBuddy 成功率 72%→90%、元宝幻觉率降 50% 等）在对抗核实中未通过，本报告不予采信。**

## 五、横向对比：国内与国际定位

### 与国内开源竞品（官方基座对比表，自报）

| 基准 | Hy3 预览版 Base | Kimi-K2 Base | DeepSeek-V3 Base | GLM-4.5 Base |
|---|---|---|---|---|
| LiveCodeBench-v6 | **34.86** | 30.86 | 29.31 | 27.43 |
| MATH | **76.28** | 71.20 | 59.37 | 61.00 |
| MMLU | 87.42（四者最末） | **88.24** | 87.68 | 87.73 |

代码和数学领先，通用知识垫底。注意：官方表述称 MMLU"略低于 Kimi 和 DeepSeek"，实际 GLM-4.5 也高于它——官方有轻微美化倾向，本报告已修正。

### 第三方独立评测（仅覆盖预览版）

- **Artificial Analysis 智能指数：34 分，93 个可比模型中排第 22**（组内平均 25 分）。官方定性为"明显高于平均水平"，其发布推文亦称 Hy3 预览版"落后于近期开源同行"。
- 第三方汇总的 SWE-bench Verified 对比：Hy3 预览版 74.4，低于 Claude Opus 4.6（80.8）、GLM-5（77.8）、Kimi-K2.5（76.8）。正式版自报 78 分若属实，可追平 GLM-5、仍落后 Claude 约 3 分——但该数字待复测。

### 与国际闭源前沿的对比（重大空白）

腾讯官方新闻稿（中英文版）**未与 GPT、Claude、Gemini 做任何直接对比**，仅与自家前代 Hy2 比较。这本身是个信号：如果对比结果有利，厂商通常不会沉默。

## 六、证据缺口与保留意见

1. **时效**：正式版发布仅一天，所有正式版成绩均为自报，第三方评测只覆盖 4 月预览版。
2. **利益相关**：幻觉率、盲测、495 步工作流等全部核心数据出自腾讯内部，270 人盲测评委是腾讯员工。
3. **社区口碑缺失**：开发者实测、论坛真实反馈未获得任何通过三票核实的证据，本报告无法评估实际使用体验。
4. **待观察问题**：正式版 78 分能否被 Artificial Analysis / LMArena 复现；495 步工作流能否在腾讯产品之外复现；本地部署后推理速度、工具调用稳定性是否与官方数据相符。

## 七、总体判断

- **新闻真实性**：属实，官方一手来源齐全，开源诚意足（Apache 2.0、权重全放）。
- **"跃升"表述**：相对自家前代 Hy2 成立；相对行业则言过其实——依据全是内部数据。
- **国内定位**：开源第一梯队偏后。代码/数学是长板，通用知识是短板；预览版基准落后于 GLM-5、Kimi-K2.5，正式版自报数字追平 GLM-5 但待验证。
- **国际定位**：与闭源前沿有明确差距（预览版 SWE-bench 落后 Claude Opus 4.6 约 6 分），官方回避直接对比。
- **真正的亮点**：21B 激活参数做到这个成绩，推理成本优势明显，适合腾讯自家海量产品内嵌落地——这与"495 步工作流""首字时延降 54%"等产品化指标的叙事方向一致。**Hy3 的战略意义在于降本增效支撑腾讯产品矩阵，而非冲击能力榜首。**
