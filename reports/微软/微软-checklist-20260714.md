## 第一关：我能理解这门生意吗（能力圈）

### 一句话说清

微软把企业的身份、办公、数据、开发、安全和云计算工作流装进同一套订阅与按量付费平台，客户因迁移成本高、协同价值大而持续付费。

| 必答问题 | 回答 | 判断 |
|---|---|---|
| 靠什么赚钱？ | Microsoft 365、Azure、服务器与安全软件收订阅费或用量费；Windows、LinkedIn、广告和游戏补充收入 | 说得清 |
| 10年后还在做什么？ | 大概率仍提供企业生产力、云、数据、安全、开发工具和 AI 平台，只是交互方式从“软件席位”转向“席位+智能体+算力消耗” | 确定性较高 |
| 哪些变量决定成败？ | Azure 增长与利用率、AI 产品 ARPU、数据中心资本回报、M365 留存、OpenAI 依赖、监管 | 可跟踪 |
| 认知来源？ | 年报、10-Q、电话会、代理声明及结构化数据交叉核验 | 非道听途说 |

微软业务线多、技术栈复杂，但收入机制清楚。它不是“卖 AI 故事”，而是把 AI 嵌入已有的企业分发渠道，再收取席位费和算力费。

**评分：★★★★☆（4/5）**

5年 ROE 轨迹为 FY2021 47.08%、FY2022 47.15%、FY2023 38.82%、FY2024 37.13%、FY2025 33.28%。绝对水平仍卓越，但权益基数扩张快于利润，方向不应忽略。

> 芒格式追问：如果一家公司利润增长 23%，但自由现金流被更快的资本开支吞掉，你到底拥有的是软件收费权，还是一座需要持续扩建的数字钢厂？

## 第三关：护城河够不够深（竞争优势）

| 护城河类型 | 是否具备 | 具体证据 | 趋势 |
|---|---|---|---|
| 品牌/定价权 | 是 | FY26 Q3 M365 Commercial Cloud 收入 +19%；Copilot 带动 ARPU | 变宽，但需证明 AI 价格可持续 |
| 转换成本 | 很强 | Entra/Active Directory、Office 文件格式、Teams、Dynamics、Power Platform、Azure 合规与运维体系相互嵌套 | 变宽 |
| 网络效应 | 中强 | GitHub 开发者生态、Windows/Office 文档协作、Marketplace 与合作伙伴渠道 | 变宽 |
| 成本/规模优势 | 很强 | 全球数据中心、企业销售渠道、研发投入和对算力采购的规模优势 | 变宽，但资本密集度上升 |
| 技术/专利壁垒 | 强 | Azure 全栈、GitHub、数据库、安全、模型平台与自研模型/芯片协同 | 稳定至变宽 |

FY26 Q3 的关键证据：Azure 及其他云服务收入 +40%；微软 AI 业务 ARR 超过 $37B、同比 +123%；Microsoft 365 Copilot 付费席位超过 2,000 万。云市场仍由 AWS、Azure、Google Cloud 三强主导，Azure 是全球第二大基础设施云平台之一。

**100亿美元复制测试：不能。** 资金可以买服务器，却买不到数十年的企业身份关系、Office 文档标准、GitHub 生态、全球合规认证和成千上万家渠道伙伴。竞争者可以复制某个产品，无法在短期内复制整个分发系统。

护城河最大的潜在破坏者不是一家普通软件公司，而是三种结构性变化：AI 智能体绕过传统应用入口；开源模型把推理价格压成商品；监管迫使微软拆分捆绑销售或开放数据接口。

**评分：★★★★★（5/5）**

## 关键风险与跟踪指标

| 风险 | 概率判断 | 影响 | 可观察指标 |
|---|---|---|---|
| AI capex 回报低于预期 | 中 | 高 | FCF/净利润持续 <60%；云毛利率继续下降；GPU 利用率不足 |
| OpenAI 集中与合作关系变化 | 中 | 高 | FY26 Q2 约45%商业 RPO 来自 OpenAI；合同、算力采购或模型授权变化 |
| Azure 被 AWS/GCP/自建算力压制 | 中低 | 高 | Azure 增速连续两个季度低于市场；新增客户与份额停滞 |
| Copilot 付费转化不及预期 | 中 | 中高 | 付费席位增长、ARPU、续费率、推理成本 |
| 反垄断与捆绑销售监管 | 中 | 中高 | 欧盟/美国对 Teams、Azure、AI 分发的补救措施 |
| 网络安全或重大服务中断 | 低 | 极高 | 安全事故、客户流失、监管处罚、SLA赔付 |

**最强空头论点：** 市场仍把微软当高毛利软件公司定价，但公司正用接近全年收入六成的 2026 日历年 capex 指引争夺 AI 产能；若推理价格持续下跌、OpenAI 采购承诺弱化，折旧和租赁成本会留下，FCF 却回不来。23倍 PE 看似便宜，40倍 FCF 才是真相。

**最可能的分析错误：** 把当前 capex 全部视为维护性支出，从而低估未来 FCF；或反过来把它全部视为浪费，从而低估供给约束下 Azure 需求和未来收入。真实答案取决于未来三年的利用率与定价。
