# 建议跟踪台账 Implementation Plan

> **现称：买卖建议跟踪表**（`reports/买卖建议跟踪表.md`；旧称「建议跟踪台账」）。

> **For agentic workers:** Execute task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 落地 `reports/买卖建议跟踪表.md`（原计划名「建议跟踪台账」），回填全部 `最终报告.md` 中的买卖建议，并在重点标的看板增加摘要。

**Architecture:** 单页活文档台账（按建议事件一行）+ 看板摘要链；首版手动维护；升格规则见设计文档。

**Tech Stack:** Markdown；仓库 `reports/`、`docs/`、`CLAUDE.md`。

## Global Constraints

- 设计依据：`docs/2026-07-20-建议跟踪台账-design.md`
- 回填范围：全部 `reports/**/最终报告.md`（约 43 份）
- 动作默认取**稳健型**建议；无分层则取报告最终结论
- 无明确买卖/观望/回避建议的报告：跳过并在「未收录说明」列出
- 命中率分母仅含 `命中`+`未命中`；历史回填默认状态 `待触发` 或据文意 `已关闭`+结果（仅当报告已自带事后结论时）
- 语言：中文；不复制长文；来源用相对路径
- 不主动 git commit（除非用户另行要求）
