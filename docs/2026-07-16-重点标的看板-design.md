# 重点标的看板 — 设计说明

**日期**：2026-07-16  
**状态**：首版已落地（`reports/重点标的看板.md`，2026-07-16）  
**目标**：在一份活文档中快速查看「有 thesis / 指定」标的的研究进度、论文健康度与待关注事项。

---

## 1. 问题与目标

`reports/` 下公司与主题报告较多，打开某家公司才能看到细节。需要一层跨公司总览，回答：

1. 这家做到哪一步了？（研究阶段、最新报告）
2. 投资论文还好吗？（假设/红线状态）
3. 这周要盯什么、下一步做什么？

本看板是**入口与状态灯**，不是第二套研究报告。

---

## 2. 方案选择

采用**单页活文档**（持续覆盖更新）：

| 对比项 | 本方案 | 未采用 |
|--------|--------|--------|
| 文件形态 | `reports/重点标的看板.md` 一份 | 周度快照多份（主题监控式） |
| 维护 | 手动，只改有变化的行 | Skill/脚本半自动（可后加） |
| 历史 | 依赖 git | 日期文件快照 |

未采用「总览表 + 公司卡片两层」，避免手动维护两处漂移。

---

## 3. 文件位置与边界

| 项 | 约定 |
|----|------|
| 看板文件 | `reports/重点标的看板.md` |
| 收录规则 | 扫描全部 `reports/**/*-thesis*.md` 自动进表；额外指定标的可手加一行（即使尚无 thesis） |
| 不收录 | 仅有 research/team、无 thesis、也未指定的公司 |
| 与 thesis | 看板摘要；假设表、红线、估值锚点仍在 `reports/{公司}/{公司}-thesis.md` |
| 与 portfolio-latest | 看板不写仓位占比；组合层仍由 `/portfolio-review` → `portfolio-latest.md` |
| 与主题监控 | 主题监控管赛道 L0；本看板管**个股**重点标的，不混表 |

原则：细节一律用相对路径链回公司目录，看板不复制长文。

---

## 4. 表结构

### 4.1 页眉（必填）

- 数据截止日
- 上次更新
- 收录规则一句话（指向本节约定即可）

### 4.2 主表字段（每行一家公司）

| 字段 | 写法 | 说明 |
|------|------|------|
| 标的 | 公司名 + 代码（可选） | 主键 |
| 研究阶段 | `未开` / `research` / `team` / `earnings` / `thesis` / `持续跟踪` | 取当前最高水位；可写 `thesis+earnings` |
| 最新报告 | 相对路径或文件名 + 日期 | 该公司目录最新一份实质报告 |
| Thesis 健康度 | `🟢完整` / `🟡需关注` / `🔴触发重审` / `⚪无thesis` | 据 thesis 假设与红线判断；指定但无 thesis 用 ⚪ |
| 假设摘要 | 一句话 | 如「7 条中 6 成立，1 待验，无红线」；细节回 thesis |
| 本周关注 | 一句话待办或风险 | 无则 `—` |
| 下次动作 | 可执行下一步 | 如「等 Q2 财报」「补 checklist」；无则 `—` |
| 链接 | thesis + 最新 research 相对路径 | 跳转用 |

### 4.3 页脚：待办汇总

只列：

- 健康度为 🟡 或 🔴 的行
- 「下次动作」不为 `—` 的行

便于扫异常，无需通读全表。

---

## 5. 健康度判定（人工约定）

| 灯 | 条件（满足即可标） |
|----|-------------------|
| 🟢 | 核心假设多数成立；无红线触发；最近一次检查有记录 |
| 🟡 | 有待验/弱化假设；或关键数据过旧需复核；或出现需跟踪的负面信号但未触红线 |
| 🔴 | thesis 红线任一条触发，或「必须重新评估」条件已满足 |
| ⚪ | 指定观察标的，尚无 `*-thesis.md` |

不确定时标 🟡，并在「本周关注」写「待复核」，禁止用推测填 🟢。

---

## 6. 维护流程

### 6.1 日常（建议每周 1 次，约 15–30 分钟）

1. 扫 `reports/**/*-thesis*.md`，有新 thesis → 加一行  
2. 只更新有变化的行（阶段、最新报告、健康度、本周关注、下次动作）  
3. 无变化：本周关注可写「较上周无新增」或保持 `—`  
4. 更新页眉「上次更新」；刷新页脚待办汇总  

### 6.2 明确不做

- 不在看板重写估值与长逻辑  
- 不按日期另存看板快照（历史用 git）  
- 首版不自动拉股价、不强制挂新 skill（后续可选）  

---

## 7. 首版落地步骤

1. 新建 `reports/重点标的看板.md`（含页眉说明 + 空/初填主表 + 待办汇总）  
2. 按现有 thesis 填入首批（至少：腾讯、拼多多、美团、泡泡玛特、微软、SK海力士）  
3. 健康度与「本周关注」据各 thesis 现状填一版；不确定 → 🟡 +「待复核」  
4. （可选）在 `CLAUDE.md` 报告命名表增加一行，指向本看板  

首批收录以落地时仓库内实际 `*-thesis*.md` 扫描结果为准。

---

## 8. 非目标与后续可选

**非目标（本设计不做）**

- 替代 `/thesis-tracker` 或 `/portfolio-review` 的完整流程  
- 主题赛道周度监控（仍用 `reports/主题监控/`）  
- 自动生成/脚本刷新（手动优先）  

**后续可选（未承诺）**

- `/focus-board` skill：按本表字段协助刷新一版  
- 脚本：仅列出「有 thesis 但不在看板」的漏网公司  

---

## 9. 验收标准

- 打开 `reports/重点标的看板.md`，能在一页内看到全部 thesis 标的的阶段、健康度、本周关注与下次动作  
- 从链接能跳到对应 thesis / 最新报告  
- 页脚待办汇总与主表 🟡/🔴/有下次动作一致  
- 看板正文不出现大段估值复制，细节在公司目录  

---

## 10. 已确认决策摘要

| 决策点 | 结论 |
|--------|------|
| 内容 | 研究进度 + 论文健康度 |
| 收录 | 全部 thesis 自动收录 + 可指定 |
| 更新方式 | 手动维护 |
| 形态 | 单页活文档 |
| 设计文档路径 | `docs/2026-07-16-重点标的看板-design.md` |
